1. Project Overview / 프로젝트 개요
1.1 Pikko is a mobile-first sports participation and booking platform for the Korean market. The initial product focus is pickleball in Seoul, with architecture designed to expand into badminton, futsal, tennis, and other sports.
1.2 The purpose of this Agreement is to engage the Developer to transform the existing frontend prototype into a production-ready MVP, including backend architecture, database design, payment integration, and live deployment.
1.3 A functional React-based frontend prototype with Kakao Map integration and mock data layer already exists. The Developer will build upon this foundation.
2. Scope of Work / 작업 범위

The Developer agrees to deliver all of the following components as part of this engagement:

Component Description
2.1 Frontend Refactor and stabilize existing React prototype; ensure mobile-first responsive design; integrate Kakao Map SDK on Home screen; connect all UI components to backend APIs; implement smooth navigation and loading states.
2.2 Backend API Design and implement a RESTful API layer; user authentication (email/password + Kakao OAuth ready); game creation and joining logic; court booking engine; operator/admin endpoints. API must be documented.
2.3 Database Design and implement database schema for: Users, Facilities, Games, Bookings, Payments. Must support soft deletes, timestamps, and be structured for future multi-sport expansion.
2.4 Payment Integrate KakaoPay (primary) or equivalent Korean payment processor. If merchant approval is pending, implement a polished demo payment flow. Backend must expose /payment/ready and /payment/approve endpoints compatible with KakaoPay v1 API.
2.5 Deployment Deploy frontend to Vercel, Firebase Hosting, or equivalent. Deploy backend to AWS, GCP, Railway, or equivalent. Configure custom domain and SSL. Ensure production stability and uptime monitoring.
2.6 Security Secure all API endpoints with authentication middleware. No API keys or secrets exposed on the frontend. Implement role-based authorization (user / operator / admin). Prepare for Korean Personal Information Protection Act (PIPA / 개인정보보호법) compliance.
2.7 Admin Dashboard Basic web interface for operators to: manage court listings, monitor bookings, view game activity, and enable/disable facilities. Accessible only to admin-role users.
