프로젝트명 Pikko — 한국 스포츠 참여 및 예약 플랫폼 (MVP)
계약금액 ₩10,000,000 원 (약 USD 7,500)
개발 기간 계약 체결일로부터 6~8주
준거법 대한민국 법률
제1조. 프로젝트 개요
1.1 Pikko는 대한민국 시장을 대상으로 한 모바일 우선 스포츠 참여 및 코트 예약 플랫폼입니다. 초기 서비스는 서울 지역 피클볼(Pickleball)에 집중하며, 향후 배드민턴, 풋살, 테니스 등 다양한 종목으로 확장할 수 있는 아키텍처를 기반으로 합니다.
1.2 본 계약의 목적은 을(개발자)이 기존의 프론트엔드 프로토타입을 기반으로 백엔드 아키텍처, 데이터베이스 설계, 결제 연동, 실서버 배포를 포함한 운영 가능한 MVP를 개발하여 납품하는 데 있습니다.
1.3 현재 카카오맵 연동 및 목업 데이터 레이어를 포함한 React 기반 프론트엔드 프로토타입이 존재합니다. 을은 이를 기반으로 개발을 진행합니다.
제2조. 작업 범위

을은 본 계약의 일환으로 아래의 모든 구성 요소를 개발하여 납품할 의무가 있습니다.

구성 요소 상세 내용
2.1 프론트엔드 기존 React 프로토타입 리팩토링 및 안정화; 모바일 우선 반응형 UI 구현; 홈 화면 카카오맵 SDK 연동; 모든 UI 컴포넌트를 백엔드 API에 연결; 부드러운 화면 전환 및 로딩 상태 구현.
2.2 백엔드 API RESTful API 설계 및 구현; 이메일/비밀번호 인증 + 카카오 OAuth 연동 준비; 게임 생성 및 참여 로직; 코트 예약 엔진; 운영자/관리자용 엔드포인트 개발. API 문서화 필수.
2.3 데이터베이스 사용자(Users), 시설(Facilities), 게임(Games), 예약(Bookings), 결제(Payments) 테이블 설계 및 구현. 소프트 삭제, 타임스탬프 지원, 향후 다종목 확장을 고려한 구조로 설계.
2.4 결제 연동 카카오페이(KakaoPay) 또는 동급 국내 결제 수단 연동(1순위). 가맹점 승인 대기 중일 경우 데모 결제 플로우 구현. 백엔드는 카카오페이 v1 API 호환 /payment/ready, /payment/approve 엔드포인트를 제공해야 함.
2.5 배포 및 인프라 프론트엔드: Vercel, Firebase Hosting 또는 동급 서비스 배포. 백엔드: AWS, GCP, Railway 또는 동급 서비스 배포. 커스텀 도메인 및 SSL 설정. 프로덕션 안정성 및 가동률 모니터링 구성.
2.6 보안 모든 API 엔드포인트에 인증 미들웨어 적용. 프론트엔드 상 API 키 및 시크릿 키 노출 금지. 역할 기반 권한 관리(사용자/운영자/관리자) 구현. 개인정보보호법(PIPA) 대응 준비.
2.7 관리자 대시보드 운영자용 기본 웹 인터페이스: 코트 목록 관리, 예약 현황 조회, 게임 활동 모니터링, 시설 활성화/비활성화 기능. 관리자 권한 계정으로만 접근 가능.
제3조. 납품물
3.1 공개 URL로 접속 가능한 완전히 작동하는 실서버 배포 MVP.
3.2 비공개 Git 저장소(GitHub 또는 동급)를 통해 납품되는 전체 소스 코드. 최종 대금 수령 시 갑(발주자)에게 소유권 이전.
3.3 백엔드 API 문서 (Postman 컬렉션, Swagger 또는 동급).
3.4 결제 확인 및 QR 입장권 생성이 포함된 결제 플로우 (카카오페이 실연동 또는 데모).
3.5 운영자 계정으로 접근 가능한 관리자 대시보드.
3.6 환경 설정 문서 (.env 구조, 배포 안내서).
3.7 출시 후 2주간의 버그 수정 지원 (추가 비용 없음).
제4조. 개발 일정
4.1 개발은 계약금(30%) 수령 및 계약서 서명 완료 시점부터 시작됩니다. 예상 개발 기간은 6~8주이며 세부 일정은 아래와 같습니다.
1~2주차
기반 구축: 백엔드 스캐폴딩, 데이터베이스 스키마 설계, 인증 시스템, API 구조 설계. 프론트엔드-백엔드 연결 설정.
3~4주차
핵심 기능 개발: 예약 엔진, 게임 생성/참여, 카카오맵 연동, 결제 엔드포인트 개발. 중간 마일스톤 검토.
5~6주차
통합 및 테스트: 결제 플로우 완성(카카오페이 또는 데모), 관리자 대시보드, QR 입장권 생성, 전체 흐름 테스트.
7~8주차
마무리 및 배포: 보안 강화, 성능 최적화, 프로덕션 배포, 도메인+SSL 설정, 문서 납품.
4.2 사전 서면 통보를 전제로 마일스톤당 최대 1주의 일정 연장이 허용되며, 총 연장 기간은 재협의 없이 최대 2주를 초과할 수 없습니다.
제5조. 대금 지급 조건
회차 마일스톤 지급 시점 금액 (원) 금액 (USD)
1차 계약 체결 계약서 서명 완료 즉시 ₩3,000,000 ~$2,250
2차 중간 납품 백엔드 API·DB·인증 라이브 상태 확인 후 (4주차 검토 승인) ₩4,000,000 ~$3,000
3차 최종 납품 MVP 전체 배포·테스트 완료 및 갑 승인 후 ₩3,000,000 ~$2,250
합계 ₩10,000,000 ~$7,500
5.1 대금은 각 마일스톤 달성 확인 후 영업일 기준 3일 이내에 계좌이체로 지급합니다. USD 금액은 참고치이며 실제 환율에 따라 달라질 수 있습니다.
5.2 최종 대금 지급은 납품물 전체에 대한 수락을 의미합니다. 갑이 최종 검토 시 이의를 제기하는 경우, 을은 7일 이내에 해당 사항을 수정한 후 최종 대금을 청구할 수 있습니다.
제6조. 지적재산권
6.1 최종 대금 수령 완료 시, 소스 코드, 디자인 에셋, 데이터베이스 스키마, 문서 등 모든 지적재산권은 갑에게 완전히 이전됩니다.
6.2 최종 대금 지급 전까지 납품물에 대한 소유권은 을에게 있습니다. 갑은 최종 대금 수령 전까지 납품물을 상업적 목적으로 배포하거나 서브라이선스할 수 없습니다.
6.3 을은 갑의 서면 요청이 없는 한, 본 프로젝트의 명칭, 카테고리, 기술 스택만을 포트폴리오에 명시할 수 있습니다.
제7조. 비밀유지
7.1 양 당사자는 프로젝트의 세부 내용, 사업 전략, 기술 구현 방식, API 키, 사용자 데이터, 재무 조건 등 모든 기밀 정보를 엄격히 비밀로 유지하며, 사전 서면 동의 없이 제3자에게 공개하지 않습니다.
7.2 본 의무는 계약 종료 후 2년간 유효합니다.
7.3 을은 갑으로부터 제공받은 접근 자격증명, 사용자 데이터, API 키를 본 계약의 이행 이외의 목적으로 사용할 수 없습니다.
제8조. 보증 및 하자 보수
8.1 을은 납품물이 본 계약서에 기술된 바와 같이 작동하고, 알려진 중대한 버그가 없으며, 최종 납품 시점에 정상적인 사용 환경에서 안정적으로 동작할 것을 보증합니다.
8.2 을은 최종 납품 후 2주간 무상 버그 수정 지원을 제공합니다. 이는 을의 구현에 직접적으로 기인한 결함에 한합니다.
8.3 하자 보수의 범위에는 신규 기능 요청, 제3자 API 정책 변경(예: 카카오 정책 변경), 갑의 코드 수정으로 인한 문제는 포함되지 않습니다.
제9조. 계약 해지
9.1 양 당사자는 상대방에게 서면으로 14일 전 통지함으로써 본 계약을 해지할 수 있습니다.
9.2 계약 해지 시, 갑은 해지일까지 완료·인수된 작업에 대해 총 계약금액 기준 비례 금액을 지급합니다.
9.3 갑이 중간 마일스톤 대금 지급 후 정당한 사유 없이 계약을 해지하는 경우, 해당 대금은 환불되지 않습니다.
9.4 을이 제4조에 명시된 일정(허용 연장 포함) 내에 합의된 마일스톤을 납품하지 못하는 경우, 갑은 계약을 해지하고 미납품 작업에 대해 지급된 대금의 비례적 환불을 요구할 수 있습니다.
제10조. 분쟁 해결
10.1 양 당사자는 분쟁 발생 시 서면 통지일로부터 14일 이내에 성실한 협의를 통해 우선 해결을 시도합니다.
10.2 협의가 성립되지 않는 경우, 대한민국 서울에서 양 당사자가 합의한 조정인을 통해 조정을 신청합니다.
10.3 조정이 성립되지 않는 경우, 분쟁은 대한민국 서울 법원의 전속 관할로 하며, 대한민국 법률을 준거법으로 합니다.
제11조. 일반 조항
11.1 완전 합의. 본 계약서는 양 당사자 간의 완전한 합의를 구성하며, 동 주제에 관한 이전의 모든 협의, 진술 또는 합의에 우선합니다.
11.2 변경. 본 계약의 수정 또는 변경은 양 당사자가 서면으로 서명한 경우에만 유효합니다.
11.3 독립 계약자. 을은 독립 계약자로서 갑의 직원이 아닙니다. 을은 자신의 세금, 보험, 복리후생에 대해 단독으로 책임집니다.
11.4 분리 가능성. 본 계약의 특정 조항이 집행 불가능한 것으로 판단되더라도 나머지 조항은 계속 유효합니다.
11.5 불가항력. 천재지변, 정부 조치, 제3자(예: 카카오)의 플랫폼 API 정책 변경 등 합리적인 통제 범위를 벗어난 사유로 인한 지연에 대해서는 어느 당사자도 책임을 지지 않습니다.
